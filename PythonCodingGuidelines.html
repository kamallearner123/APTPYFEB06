Python Best Practices for Readability

1. Follow PEP 8

Use proper indentation, line length, and spacing.

def greet(name: str) -> str:
    """Return a greeting message."""
    return f"Hello, {name}!"

2. Use Meaningful Variable and Function Names

Avoid single-letter names except for loop counters.

# Bad
x = 10

# Good
max_users = 10

3. Use Type Hints

def add_numbers(a: int, b: int) -> int:
    return a + b

4. Keep Functions Short and Focused

def calculate_area(width: float, height: float) -> float:
    """Calculate the area of a rectangle."""
    return width * height

5. Use Docstrings for Documentation

def square(n: int) -> int:
    """Returns the square of a number."""
    return n * n

6. Avoid Magic Numbers

# Bad
price = amount * 0.08

# Good
TAX_RATE = 0.08
price = amount * TAX_RATE

7. Use List Comprehensions

# Bad
squares = []
for num in range(10):
    squares.append(num ** 2)

# Good
squares = [num ** 2 for num in range(10)]

8. Use `with` for File Handling

with open("data.txt", "r") as file:
    content = file.read()

9. Handle Exceptions Properly

try:
    result = 10 / 0
except ZeroDivisionError as e:
    print(f"Error: {e}")

10. Use Logging Instead of Print Statements

import logging

logging.basicConfig(level=logging.INFO)
logging.info("Application started")

11. Use `enumerate()` Instead of `range(len())`

# Bad
for i in range(len(items)):
    print(i, items[i])

# Good
for index, item in enumerate(items):
    print(index, item)

12. Follow Consistent Naming Conventions

Use snake_case for functions and variables, PascalCase for classes, and UPPER_CASE for constants.

13. Use Virtual Environments

python -m venv env
source env/bin/activate  # Mac/Linux
env\Scripts\activate     # Windows

14. Use `dataclasses` for Struct-like Objects

from dataclasses import dataclass

@dataclass
class User:
    name: str
    age: int

15. Use `f-strings` Instead of `format()`

name = "Alice"
print(f"Hello, {name}!")
